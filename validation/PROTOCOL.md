# 粘贴可行性验证协议（写代码前必跑，约 30 分钟）

> 目的：用纯手工方式验证 v1 的核心产品假设——"生成的成品能粘贴/上传可用"。
> 结果直接决定产品形态，**不通过就不该按当前方案投 Phase 3/4**。

---

## 测试一：公众号富文本粘贴（最关键）

### 步骤
1. 本地起一个 https/localhost 服务（ClipboardItem 需要安全上下文）：
   ```bash
   cd validation && python3 -m http.server 8080
   ```
   浏览器打开 `http://localhost:8080/wechat-paste-test.html`。
2. 点 **「① 复制为富文本」**。
3. 打开公众号「图文编辑器」（草稿即可，不用发布），在正文区 `Cmd/Ctrl+V` 粘贴。
4. 再用 **「② 仅选中正文」** + 手动 `Cmd/Ctrl+C` 粘一次，作对照。

### 结果清单（逐项打勾）
| 检查项 | 机制① | 机制② | 备注 |
|---|---|---|---|
| 标题字号/颜色保留 | ☐ | ☐ | |
| 加粗/绿色强调保留 | ☐ | ☐ | |
| 引用块灰底+左边框保留 | ☐ | ☐ | |
| 代码块深色背景+等宽保留 | ☐ | ☐ | 这条最容易丢 |
| 行内代码底色保留 | ☐ | ☐ | |
| 列表样式正常 | ☐ | ☐ | |
| **外链**是否保留为可点链接 | ☐ | ☐ | 若被吞 → 确认要做「链接转脚注」 |
| **本地图**是否显示 | ☐ 预期✗ | ☐ | 预期失败，确认本地图必须报 warning |
| **公网图 A**（海外直链）是否转存 | ☐ | ☐ | 直链无 302，失败可归因于防盗链/host 可达性 |
| **公网图 B**（国内 CDN 直链）是否转存 | ☐ | ☐ | 与 A 对照 host 差异 |

> ⚠️ **不要用会 302 重定向的图源**（如 picsum.photos）：转存失败时无法区分"防盗链拦截"还是"爬虫没跟重定向"。两张都已换成直链。**最准的做法是把图 A 换成你真正要用的图床直链。**

### 判定
- ✅ **通过**：机制① 下样式基本保留、至少一类公网直链能转存 → 公众号走"粘贴"路线成立，v1 方案确认。
- ⚠️ **部分**：样式保留但公网图都失败 → 文档明确"图床 + 防盗链 + host 可达性"是前置条件，图片走引导清单。
- ❌ **不通过**：样式大面积丢失 → 需改方向（如借 mdnice 已验证的复制实现 / 重新评估公众号形态）。

## 测试二：小红书文案体验（确认"纯文本够不够用"）

小红书无网页粘贴富文本概念，正文是纯文本 + emoji + `#话题#`，且**图片优先**。

### 步骤
1. 准备一段口语化文案样例（手写或下面模板），含 emoji 与 2~3 个 `#话题#`。
2. 在小红书 App/网页发布页粘贴文案，**自己另配图片**，观察体验。

模板：
```
姐妹们我真的会爆哭😭 用一周搓了个多平台发布工具

之前每次发公众号+知乎+小红书要排版三遍🤡
现在一份内容自动适配各平台 真的省到飞起✨

具体怎么做的👇评论区聊
#效率工具 #程序员日常 #搞钱工具
```

### 要回答的问题
- 纯文案（无配图）对小红书用户**到底有没有价值**？还是必须配"文字卡片图"？
- 1000 字限制下，长文压缩成短文案的损耗能不能接受？

### 判定
- 若"纯文案 + 创作者自配图"可接受 → 小红书 v1 只产文案 + 话题。
- 若"没图近乎无用" → 把「文本→文字卡片图」列入 Phase 3 小红书增强（需 headless 截图渲染）。
