# 计划文档 —— MultiPub

## 1. 范围共识（已确认 ✅）

- **核心交付**：Markdown → 多平台格式/风格适配 + 模拟发布 + 可扩展架构。
- **形态**：**Web 可视化优先** —— 编辑器 + 多平台实时预览。后端 FastAPI 复用核心层，前端轻量单页。
- **平台**：公众号 / 小红书 / 知乎 / B站专栏 **四个全做**（建议公众号、小红书先行，二者差异最大，验证 IR 抽象）。
- **风格适配**：**做，作为可选模块**（LLM 改写，可一键开关，默认关回退纯格式适配）。
- **发布**：**全程模拟发布** —— 所有平台生成可直接粘贴/上传的成品 + 模拟回执，**不接任何平台 API**。
  > 好处：核心价值不依赖任何外部服务可用性，无凭证/认证/IP 白名单门槛，落地最快。
  > 真实发布（如公众号草稿 API）留作未来扩展，`Publisher` 接口已为其预留，不动核心。

## 2. 分阶段里程碑

> 每阶段都可独立演示、可验收。建议按此顺序，每阶段产出可见价值。

> **当前进度（2026-05-29）**：Phase 0 ✅ / Phase 1 ✅ / Phase 2 ✅ / Phase 3 ✅ / Phase 4 ✅，**全量 89 测试通过**。Phase 5 待办。
> 四平台（公众号/小红书/知乎/B站）+ 实时预览 + 模拟发布 + 可选 LLM 风格适配 已全部就绪。

### Phase 2 —— Web 端打通（本阶段交付可演示产品）

### Phase 4 —— 风格适配（可选模块）

### Phase 5 —— 扩展性与增强（可选）
- [ ] `entry_points` 第三方平台插件示例（写一个 demo 平台证明可扩展）
- [ ] 真实发布预留：`WeChatDraftPublisher`（未来若有认证服务号凭证再接）
- [ ] 多账号 / 批量 / 导出
- [ ] CLI 入口（复用同一核心层，零额外核心改动）

## 3. 优先级建议

**Phase 0 → 1 → 2 是核心路径**：跑到 Phase 2 就有一个可在浏览器演示的产品（写 Markdown → 实时看公众号/小红书适配 → 模拟发布）。Phase 3 补齐四平台，Phase 4（LLM 风格适配）是最出彩的加分项。Phase 5 全部可选。

## 4. 验收标准（贯穿全程）

- 同一输入产出稳定（快照测试不漂移）。
- 新增一个平台只需新增一个文件 + 注册，核心代码零改动（用 demo 平台证明）。
- 所有适配约束触发时有可见 `warnings`，不静默截断。
- 真实发布失败时错误信息可定位（区分凭证/网络/平台限制）。

## 5. 已确认决策 ✅

1. **形态**：Web 可视化优先（FastAPI 后端 + 轻量单页前端，核心层与未来 CLI 复用）。
2. **风格适配**：做，作为可选模块（LLM 改写，可开关，默认关）。
3. **发布范围**：全程模拟发布，不接平台 API；真实发布留作扩展。
4. **平台**：四个全做，公众号 + 小红书先行。
5. **技术栈**：Python（mistune 解析 + FastAPI + pydantic + anthropic 可选），前端轻量单页（零构建步骤）。
